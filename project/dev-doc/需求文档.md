# **项目需求文档：公考职位快筛助手**

## **1. 项目概述**
- **项目名称**： 公考职位快筛助手
- **项目愿景**： 为公考考生提供一个零部署、高隐私、体验流畅的工具，通过智能化的专业匹配，瞬间从海量职位中锁定目标，让报考准备事半功倍。
- **核心价值**：
    1.  **极简操作**： 上传文件，选择专业，一步获取结果。
    2.  **智能匹配**： 不仅支持精准匹配，更理解专业类别，避免遗漏。
    3.  **隐私安全**： 所有数据在用户浏览器本地完成处理，绝不外传。
    4.  **全端适配**： 在手机、平板、电脑上均能获得完美体验。
- **技术基调**： 响应式单页Web应用（SPA），纯前端技术栈实现。

## **2. 详细功能需求**

### **FR1： 文件上传与解析模块**
- **FR1.1**： 提供拖拽区域和“点击上传”按钮，支持上传 `.xls` 和 `.xlsx` 格式的Excel文件。
- **FR1.2**： 文件上传后，使用前端库在内存中解析文件内容，不产生任何服务器网络请求。
- **FR1.3**： 解析成功后，在界面以简洁的表格形式**预览前5行数据**，并高亮显示表头行，让用户确认文件内容及结构正确。
- **FR1.4**： 提供“重新上传”按钮，允许用户更换文件。

### **FR2： 智能专业筛选器模块**
- **FR2.1**： 文件解析后，自动提取“专业要求”列的所有**唯一值**，生成一个**可搜索的下拉选择器（Combobox）**。
- **FR2.2**： 用户行为：
    - **选择**： 用户可以从下拉列表中选择一个已存在的专业。
    - **键入**： 用户也可以直接输入任何文本作为筛选关键词。
- **FR2.3**： **核心匹配逻辑（模糊匹配）**：
    1.  **关键词切分**： 将用户输入或选择的内容按中文分号、逗号进行切分，得到关键词列表（如“新闻传播学类；汉语言文学”）。
    2.  **双向包含匹配**：
        - **方向A（岗位专业包含用户关键词）**： 筛选出“专业要求”单元格文本**包含**任一关键词的岗位。例如，岗位要求为“新闻传播学类、历史学”，关键词为“新闻学”，则匹配。
        - **方向B（用户关键词包含岗位专业类别）**： 设计一个简易的**专业类别映射关系**（例如，“新闻传播学类”映射到“新闻学”、“传播学”等）。当用户输入具体专业（如“新闻学”）时，若岗位专业要求为对应的上级类别（如“新闻传播学类”），则该岗位也应被匹配。
        - *（实现建议：维护一个键值对映射表，匹配时先检查直接包含，再检查通过映射表的间接包含。）*
- **FR2.4**： 提供“开始筛选”按钮触发操作，并提供“清空筛选”按钮重置筛选条件和结果。

### **FR3： 结果展示模块**
- **FR3.1 - 结果概览**： 筛选后，顶部显示“共找到 **X** 个匹配岗位”的统计信息。
- **FR3.2 - 列表视图（默认）**： 以响应式卡片列表形式展示匹配的岗位。
    - **移动端**： 每行一个卡片，显示核心信息：`[职位名称]`、`[招录部门]`、`[专业要求]`。
    - **桌面端**： 可采用表格布局，显示更多列（如学历、政治面貌）。
- **FR3.3 - 详情视图**： 点击列表中的任一岗位卡片，以抽屉（Drawer）或模态框（Modal）形式从右侧（移动端底部）滑出，展示该岗位的**所有字段信息**。
- **FR3.4**： 结果列表应支持按某些字段（如部门、职位代码）进行简单排序。

### **FR4： 响应式与辅助功能**
- **FR4.1**： 界面必须完美适配从手机（≥320px）到桌面（≥1200px）的各种屏幕尺寸。
- **FR4.2**： 确保足够的颜色对比度，支持键盘导航，遵循基础的WAI-ARIA标准，保证可访问性。

## **3. 非功能需求**
- **NFR1： 性能**： 在主流中端手机浏览器上，解析并筛选一份5000行、20列的Excel文件，总耗时不超过5秒。
- **NFR2： 可用性**： 首次使用用户无需指导，能在30秒内成功完成一次筛选操作。
- **NFR3： 兼容性**： 支持最新两个版本的Chrome, Firefox, Safari, Edge。
- **NFR4： 隐私**： 100%离线运行，代码中需明确注释并向用户提示此点，建立信任。

## **4. UI/UX 设计规范**
- **色彩体系**：
    - **主色**： `#f97316`（活力橙）
    - **辅助色**： `#fdba74`（浅橙）， `#ea580c`（深橙）
    - **背景色**： `#fff7ed`（暖白）， `#fef3c7`（用于高亮）
    - **文字色**： `#1f2937`（标题深灰）， `#4b5563`（正文灰）
    - **成功色**： `#10b981`， **警告/错误色**： `#ef4444`
- **字体**： 系统字体栈优先，`-apple-system, BlinkMacSystemFont, ‘Segoe UI’, Roboto, ‘Helvetica Neue’, Arial`。
- **布局与组件**：
    - 采用卡片（Card）、抽屉（Drawer）、悬浮按钮（FAB）等现代组件。
    - 圆角统一为 `0.75rem`，阴影使用轻量级的 `sm` 和 `md`。
    - 上传区域需有明确的视觉反馈（拖拽时高亮为 `#fdba74`）。
- **交互反馈**： 所有操作必须有即时反馈（如按钮按下态、加载中旋转动画、结果出现淡入动画）。

## **5. 数据与逻辑规则**
- **专业类别映射表**： 项目内置一个可维护的JSON对象，定义常见专业大类与具体专业的映射关系，作为模糊匹配的依据。此表应易于后续增补。
    ```json
    {
      "新闻传播学类": ["新闻学", "传播学", "广播电视学", "广告学", "编辑出版学"],
      "计算机类": ["计算机科学与技术", "软件工程", "网络工程", "信息安全"],
      // ... 更多类别
    }
    ```

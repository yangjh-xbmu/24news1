# **技术任务拆解清单**

## **阶段一：项目初始化与环境搭建 (预计: 0.5天)**
- [ ] **T1.1**： 使用Vite创建React + TypeScript项目。
- [ ] **T1.2**： 集成Tailwind CSS及PostCSS插件，并配置设计系统中的颜色、字体等主题变量。
- [ ] **T1.3**： 安装并配置核心依赖库：`xlsx` (SheetJS), `@tanstack/react-table`, `@headlessui/react` (用于Combobox、Dialog等)。
- [ ] **T1.4**： 配置Git仓库，设置合理的 `.gitignore`。
- [ ] **T1.5**： 创建基础项目结构，划分组件目录（`components/`）、工具函数目录（`lib/`）、类型定义目录（`types/`）。

## **阶段二：核心数据处理模块开发 (预计: 1.5天)**
- [ ] **T2.1**： 创建 `lib/excelParser.ts`。
    - 实现使用 `xlsx` 库读取文件并转换为JSON数组的函数。
    - 编写函数用于从数据中提取表头和特定列的唯一值。
- [ ] **T2.2**： 创建 `lib/matchLogic.ts`。
    - 定义 `ProfessionalMap` 类型和专业映射表 `professionMap`。
    - 实现核心的 `fuzzyMatch` 函数，包含关键词切分、双向包含匹配逻辑。
    - 编写单元测试（使用Vitest），验证匹配逻辑的准确性。

## **阶段三：核心UI组件开发 (预计: 2天)**
- [ ] **T3.1**： 创建 `FileUploadZone` 组件。
    - 实现拖拽和点击上传UI。
    - 集成`excelParser`，实现文件解析和预览功能。
    - 包含“重新上传”逻辑。
- [ ] **T3.2**： 创建 `ProfessionCombobox` 组件。
    - 基于 `@headlessui/react` 的 `Combobox` 实现。
    - 接收专业列表作为输入，支持搜索和选择。
    - 允许自由输入。
- [ ] **T3.3**： 创建 `ResultTableCard` 组件。
    - 使用 `@tanstack/react-table` 构建响应式表格。
    - 实现移动端卡片视图和桌面端表格视图的切换（通过CSS媒体查询或状态）。
- [ ] **T3.4**： 创建 `PositionDetailDrawer` 组件。
    - 基于 `@headlessui/react` 的 `Dialog` 实现抽屉效果。
    - 接收单条岗位数据，格式化展示所有字段。

## **阶段四：页面集成与状态管理 (预计: 1天)**
- [ ] **T4.1**： 创建主页面 `App.tsx` 或 `HomePage.tsx`。
    - 使用React Hooks (`useState`) 管理应用状态：原始数据、筛选关键词、结果数据、加载状态等。
    - 编排组件顺序：`FileUploadZone` -> `ProfessionCombobox` -> 控制按钮 -> `ResultTableCard`。
- [ ] **T4.2**： 将各个组件连接起来，实现完整的用户操作流。
    - 上传成功 -> 解析数据 -> 提取专业列表 -> 填入 `ProfessionCombobox`。
    - 点击筛选 -> 调用 `matchLogic` -> 更新结果状态 -> 触发 `ResultTableCard` 重绘。
    - 点击行 -> 打开 `PositionDetailDrawer` 并传入对应数据。

## **阶段五：响应式细化、动效与打磨 (预计: 1天)**
- [ ] **T5.1**： 为所有组件编写详细的响应式CSS，确保从手机到桌面的无缝适配。
- [ ] **T5.2**： 添加微交互动效（使用Tailwind CSS的 `transition` 和 `animation`），如按钮反馈、加载旋转、列表淡入。
- [ ] **T5.3**： 全面测试Chrome/Firefox/Safari移动端和桌面端的兼容性。
- [ ] **T5.4**： 添加明确的用户提示，例如“数据仅在本地处理，安全无忧”。
- [ ] **T5.5**： 性能优化检查（如对大列表使用虚拟滚动 `react-virtual` 的可行性评估）。

## **阶段六：构建、部署与文档 (预计: 0.5天)**
- [ ] **T6.1**： 运行生产环境构建 (`npm run build`)，检查并优化构建产物。
- [ ] **T6.2**： 准备一个简单的 `README.md`，说明项目功能、技术栈和本地运行方法。
- [ ] **T6.3**： 将构建的 `dist` 目录部署至任意静态网站托管服务（如Vercel, Netlify, GitHub Pages）。
